# Save Your Token - Coding & Workflow Standards

## Core Principles
1. Defensive Coding: Always plan, verify current environment, and read relevant files before writing code.
2. Minimalism & Efficiency: Modify only what is necessary.
3. Debugging Discipline: Prioritize investigation (logs/state) over guessing.
4. Autonomous Execution: Execute, don't just plan. Use tools to verify.

## Development Workflow
- Local Development -> Git Sync -> GitHub Update -> Verify.
- Strict token efficiency: minimize verbosity in tool calls and outputs.
